# 凯西:

*   **凯西规范**:凯西语言及其标准库的参考文档。
*   **凯西转 LLVM 编译器**:可以将凯西代码交叉编译成可以直接运行的 x86/arm/etc 代码的编译器。
*   **凯西转 FPGA 编译器**:可以将凯西代码交叉编译成可以直接在 FPGA 上加载运行的 Verilog 代码的编译器。
*   **凯西开发环境**:促进程序员友好地创建凯西代码的开发环境。*未来延期*。

# 凯西图书馆

*   凯西图书馆文件:凯西图书馆的参考文件。
*   **凯西标准库**:标准凯西函数的库。
*   **凯西 x86 支持库**:一个将一些标准的凯西函数映射到更高效的 x86 等价物的库。这些函数大多是低级函数，可以使用特定的 x86 指令进行优化。*未来扩展*。
*   **凯西 FPGA 支持库**:一个将一些标准凯西函数映射到更高效的 Verilog 等价函数的库。这些大多是低级功能，可以通过使用特定的 Verilog 指令进行优化。*未来扩展。*

# 量子协议

*   **Qubic 规范**:qu BIC 协议的参考文件。
*   Qubic 数学论文:一份类似白皮书的文档，探索了 Qubic 协议某些部分的数学特性。
*   **Qubic PoC** :运行在 Tangle 上的 Qubic 协议的非平凡示例用例。

# q-纠结:

*   **Qubic 打包器**:将凯西代码打包成 Qubic 交易的打包器。
*   **时间戳**:用于 Qubic 协议各个部分的可靠时间戳的定义和实现。
*   **交易**:所有 Qubic 特定交易的定义、记录和实施。
*   **汇编**:qu BIC 协议的一部分，处理所有与汇编相关的事情。这包括通过定义装配和时期参数来创建和维护装配。
*   **法定人数**:量子协议中处理一切与法定人数相关的部分。这包括神谕承诺和揭示结果，并就定额达成共识。它还包括 qubic 所有者验证法定人数结果。
*   **奖励**:qu BIC 协议中处理与奖励相关的所有事情的部分。这包括 qubic 的所有者方面，它决定向哪个神谕支付多少钱。它还包括跟踪实际支付的 oracle 端。
*   **资源测试**:qu BIC 协议的一部分，处理与资源测试阶段相关的所有事情。这包括定义可以作为初始实现的工作证明 qubic。它还包括 Q 节点的一部分，Q 节点运行资源测试，发布结果，并就权重因子达成一致。

# Oracle 机器

*   **Q-Tangle** :分离本地化 Tangle，优化支持 Qubic 协议。Q-tangle 将只包含与 Qubic 协议相关的事务。这一部分将挂钩到 Tangle 消息流，并过滤掉 Qubic 事务。
*   **URI 处理器**:URI 处理器的实现，用于处理外部数据的访问。
*   **调度器**:处理事件的触发，调度并运行 qubics。
*   **Q 节点**:集成了运行 Qubic 协议能力的 IOTA 完整节点。
