什么/为什么是Hyperledger结构？

Hyperledger Fabric是一个开源的企业级许可分布式分类帐技术(DLT)平台，设计用于企业环境，提供了一些超越其他流行的分布式分类帐或区块链平台的关键差异化功能。

在学习 Hyperledger Fabric 时，我从阅读 Hyperledger Fabric的 概念文档开始。我的脑子里充满了新的概念，于是我一头扎进了他们的实践教程 编写你的第一个应用程序 。

它进行得并不顺利；我努力将实践步骤与概念联系起来。在这个系列中，我们将重温动手教程；这一次我们将详细探索概念和软件组件。

软件组件

在浏览文档时，一个基本的挑战是确定他们的( Hyperledger Fabric )软件和我的(操作员/开发人员)软件之间的界限；例如，他们的实践教程严重依赖于shell脚本(他们的还是我的？)

在准备撰写本系列时，我对他们的软件组件提出了一个工作假设(如果我了解到其他情况，以后会修改)。

码头工人图片

他们的服务器软件通过 Docker Hub 上的 Docker 映像交付，例如hyperledger/fabric-peer。

命令行界面(CLI)工具

fabric-samples 安装提供了许多命令行界面工具；用于控制 Docker 图像；通过配置文件或API调用。

Hyperledger Fabric SDK(NPM软件包)

Hyperledger Fabric SDK(我们将重点关注 Node.js SDK)控制 Docker 图像的方式与CLI工具非常相似。它们通过两个NPM包fabric-client和fabric-client-ca交付。

链码接口 (NPM包)

最后，还有NPM包， fabric-shim ，它提供了一种开发智能合约的机制。，chaincode，在 JavaScript 中。
