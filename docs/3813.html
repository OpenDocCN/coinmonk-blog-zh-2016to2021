将Hyperledger Explorer与Hyperledger Fabric Network v2.2集成

1。概述

为了从数据中提取商业价值，可视化数据的能力至关重要。Hyperledger Explorer提供了这一急需的功能。Hyperledger Explorer是一个用户友好的Web应用程序工具，用于查看、调用、部署或查询块、事务和相关数据、网络信息(名称、状态、节点列表)、链代码和事务系列，以及存储在分类帐中的任何其他相关信息。在本文中，我们将学习如何将Hyperledger Explorer与Hyperledger Fabric集成。我们将使用v2.2，因为它是当前的LTS版本。

2.Hyperledger浏览器的主要功能

  • 具有丰富用户界面的Web应用程序，使用最新技术开发，如ReactJS、Google material UI、web-sockets等。
  • 可以监听和查询Hyperledger结构网络的区块链模块。
  • 获取最新的状态块、网络和链码，查看块和事务。
  • 按小时和分钟划分的块和事务度量。
  • 按日期范围和渠道搜索和过滤块、交易。
  • 动态发现新频道，并按频道切换数据显示。
  • 获得新数据块的实时通知。

3.体系结构

它由3层组成

  1. 表示层/前端-Hyperledger explorer的前端构建在ReactJs之上
  2. 后端层- 后端是建立在node js和带有WebSockets的express框架之上。
  3. 数据库层- 它使用的是 PostgreSQL 数据库。关于块、交易、通道等的信息将存储在该数据库中。

4.履行

  1. 克隆存储库
git clone https://github.com/adityajoshi12/fabric-samples-advance-topics

2.启动网络

cd fabric-samples-advance-topics/test-network
./network.sh up createChannel -c mychannel -s couchdb -i 2.2

3.启动explorer docker容器

cd ../explorer
docker-compose up -d

4.浏览器容器打开后，在浏览器上打开http://localhost:8080。您可以看到资源管理器仪表板。您需要登录，一旦成功登录，您将能够看到如下屏幕。

仪表板的用户名和密码在explorer/first-network.json中定义，为了禁用认证，我们可以将enableAuthentication设置为true

你可以用docker logs -f explorer.mynetwork.com查看日志，你可以在这里找到视频教程。

要停止hyperledger浏览器运行，请执行以下操作:

docker-compose down

5。构建自己的用户界面

您可以使用浏览器公开的REST API在hyperledger浏览器上构建自己的用户界面。Explorer团队为我们提供了Swagger规范，您可以简单地将其导入到REST客户端，如 Postman 或失眠症。你可以在这里找到swagger规格。

导入swagger规范后，您可以看到一个包含REST API的集合。

现在，您可以简单地将这些API用于您选择的任何前端框架。
